Query Chuẩn Node.js Cho PostGIS Theo Mét

Node.js Search & Location Service nên dùng đúng mẫu này để khớp với expression GiST index trong V5.

Radius quanh một tọa độ Bind $1 = longitude, $2 = latitude, $3 = radiusMeters. Giữ status = 'ACTIVE' dạng literal và location IS NOT NULL để partial index được dùng ổn định.
Index cần thấy trong EXPLAIN idx_listings_active_location_geog_gist, idx_landmarks_active_location_geog_gist hoặc idx_complexes_active_location_geog_gist tùy bảng query.
WITH q AS ( SELECT ST_SetSRID(ST_MakePoint($1, $2), 4326)::geography AS point ) SELECT l.id, l.title, l.category, l.sub_category, l.base_price, l.average_rating, l.latitude, l.longitude, ST_Distance(l.location::geography, q.point) AS distance_meters FROM public.listings l CROSS JOIN q WHERE l.status = 'ACTIVE' AND l.location IS NOT NULL AND ST_DWithin(l.location::geography, q.point, $3) ORDER BY distance_meters ASC LIMIT $4 OFFSET $5;
SELECT l.id, l.title, l.category, l.sub_category, l.base_price, l.average_rating, ST_Distance(l.location::geography, lm.location::geography) AS distance_meters FROM public.landmarks lm JOIN public.listings l ON l.status = 'ACTIVE' AND l.location IS NOT NULL WHERE lm.id = $1 AND lm.status = 'ACTIVE' AND lm.location IS NOT NULL AND ST_DWithin(l.location::geography, lm.location::geography, $2) ORDER BY distance_meters ASC LIMIT $3;
EXPLAIN (ANALYZE, BUFFERS) WITH q AS ( SELECT ST_SetSRID(ST_MakePoint($1, $2), 4326)::geography AS point ) SELECT l.id FROM public.listings l CROSS JOIN q WHERE l.status = 'ACTIVE' AND l.location IS NOT NULL AND ST_DWithin(l.location::geography, q.point, $3) ORDER BY ST_Distance(l.location::geography, q.point) ASC LIMIT $4;

Query Chuẩn Node.js Cho Autocomplete Không Dấu

Node.js nên normalize input bằng function DB để giống hệt dữ liệu đã backfill và trigger đang duy trì.

Landmark autocomplete Bind $1 = userInput, $2 = limit. Query này match được có dấu/không dấu và sai nhẹ nhờ trigram.
Index cần thấy trong EXPLAIN idx_landmarks_active_name_normalized_trgm cho Landmark, idx_listings_active_title_normalized_trgm cho Listing keyword search.
WITH q AS ( SELECT public.gostay_normalize_search_text($1) AS term ) SELECT lm.id, lm.name, lm.province, lm.latitude, lm.longitude, similarity(lm.name_normalized, q.term) AS score FROM public.landmarks lm CROSS JOIN q WHERE lm.status = 'ACTIVE' AND lm.name_normalized IS NOT NULL AND q.term IS NOT NULL AND ( lm.name_normalized LIKE q.term || '%' OR lm.name_normalized % q.term ) ORDER BY CASE WHEN lm.name_normalized LIKE q.term || '%' THEN 0 ELSE 1 END, score DESC, lm.is_featured DESC NULLS LAST, lm.name ASC LIMIT $2;
WITH q AS ( SELECT public.gostay_normalize_search_text($1) AS term ) SELECT l.id, l.title, l.category, l.sub_category, l.base_price, l.average_rating, similarity(l.title_normalized, q.term) AS score FROM public.listings l CROSS JOIN q WHERE l.status = 'ACTIVE' AND l.title_normalized IS NOT NULL AND q.term IS NOT NULL AND ( l.title_normalized LIKE '%' || q.term || '%' OR l.title_normalized % q.term ) ORDER BY score DESC, l.average_rating DESC NULLS LAST, l.title ASC LIMIT $2 OFFSET $3;

P2 - Rủi Ro Cần Theo Dõi Sau Khi P0/P1 Xong

Không nhất thiết chặn Node bắt đầu, nhưng cần đưa vào backlog để tránh dữ liệu lệch khi hệ thống lớn hơn.

P2 Đã xong
Vấn đề còn tồn đọng Review mới đã cập nhật rating, nhưng dữ liệu review cũ nếu có chưa chắc đã được backfill. Việc tính lại rating hiện đang chạy đồng bộ trong request submit review.
Tác hại Ranking/recommendation có thể sai với dữ liệu cũ. Khi traffic review tăng, request submit review bị nặng hơn.
Cần sửa
  • Migration/job backfill average_rating, total_reviews.
  • Sau MVP, chuyển sang async/event như báo cáo mô tả.
Tiêu chí hoàn thành Mọi listing có reviews đều có aggregate đúng; submit review không bị bottleneck khi nhiều request.
P2 Đã xong
Vấn đề còn tồn đọng Node có thể đọc trực tiếp nhiều table/entity của Catalog, nhưng contract đọc chưa được đóng thành view hoặc DTO DB ổn định.
Tác hại Node dễ phụ thuộc sâu vào cấu trúc Java entity. Khi Java đổi JSONB/field, Search/Recommendation có thể vỡ.
Cần sửa
  • Tạo view/materialized view listing_search_documents hoặc document hóa query contract.
  • Expose các field Node cần: id, title, category, subCategory, price, rating, location, lat/lng, status, complexId, attributes/tags.
Tiêu chí hoàn thành Node có một nguồn đọc ổn định, được grant read-only, và không cần đoán schema Java.
